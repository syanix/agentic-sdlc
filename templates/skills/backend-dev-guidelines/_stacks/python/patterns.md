# Python Backend Patterns

## Project Structure

- Use the `src` layout to separate source code from configuration and tests
- Group by feature or domain rather than by layer where practical
- Keep the application entry point thin: configure dependencies and start the server

```
src/
  app/
    main.py
    config.py
    dependencies.py
    users/
      router.py
      service.py
      repository.py
      schemas.py
      models.py
    orders/
      router.py
      service.py
      schemas.py
tests/
  users/
    test_service.py
    test_router.py
```

---

## Dependency Injection

- Use FastAPI's `Depends()` for automatic injection of services, repositories, and database sessions
- Create factory functions that return configured dependencies
- Chain dependencies: a service depends on a repository, which depends on a session
- Override dependencies in tests using `app.dependency_overrides`

```python
def get_user_service(
    repo: UserRepository = Depends(get_user_repository),
) -> UserService:
    return UserService(repo)
```

---

## Pydantic Models

- Separate request schemas from response schemas: `UserCreate`, `UserResponse`
- Use `model_validator` and `field_validator` for complex validation rules
- Define `BaseSettings` subclasses for configuration with environment variable loading
- Enable `model_config = ConfigDict(strict=True)` for strict type coercion
- Use `Field(...)` for constraints: `min_length`, `max_length`, `ge`, `le`
- Reuse common fields via model inheritance or composition

## SQLAlchemy 2.0

- Use the 2.0 style with `select()`, `insert()`, `update()` statements
- Configure `async_sessionmaker` with `AsyncSession` for async database access
- Implement the repository pattern: one repository per aggregate root
- Use Unit of Work pattern by committing at the service layer, not the repository
- Define models using `DeclarativeBase` with `Mapped` type annotations

```python
async def find_by_id(self, session: AsyncSession, user_id: UUID) -> User | None:
    result = await session.execute(select(User).where(User.id == user_id))
    return result.scalar_one_or_none()
```

## Alembic Migrations

- Use `alembic revision --autogenerate -m "description"` to generate migrations
- Configure a naming convention for constraints to ensure consistent names across environments
- Always review autogenerated migrations before committing; adjust as needed
- Run `alembic upgrade head` as part of the deployment pipeline
- Use `alembic downgrade -1` for rollback; ensure every migration is reversible

---

## Error Handling

- Define custom exception classes inheriting from a base `AppException`
- Register FastAPI exception handlers to map domain exceptions to HTTP responses
- Return consistent error response shapes with `detail`, `code`, and optional `errors` array
- Use `HTTPException` only in routers, never in services or repositories
- Log unhandled exceptions with full traceback and request context

---

## Async Patterns

- Use `async def` for all I/O-bound route handlers and service methods
- Use `asyncio.gather()` for concurrent independent operations
- Avoid `run_in_executor` unless interfacing with synchronous libraries
- Use `BackgroundTasks` for fire-and-forget operations (email, notifications)
- Be cautious with shared mutable state across concurrent tasks

---

## Type Hints

- Enable strict mode in `mypy.ini` or `pyproject.toml`: `strict = true`
- Use `Protocol` classes to define interfaces without inheritance
- Annotate all function signatures, including return types
- Use `TypeVar` and `Generic` for reusable typed patterns (e.g., generic repository)
- Prefer `X | None` over `Optional[X]` for union types
